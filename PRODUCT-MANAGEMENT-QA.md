# Product Management — Manual QA Checklist

> محدوده: بخش مدیریت محصولات (ADMIN-CP3 / 3.1 / 3.2 / 3.3). همه‌ی مسیرها نیازمند `role=ADMIN`.
> قیمت‌ها: ورودی **تومان**، ذخیره **ریال** (ریال = تومان × ۱۰). نامک/SKU فقط لاتین.

## مسیرها
- `/admin/collections/products` — لیست
- `/admin/collections/products/new` — فرم کامل
- `/admin/collections/products/[id]` — ویرایش + Variant Matrix + حذف
- `/admin/collections/products/quick-add` — شیت سریع
- `/admin/collections/weights-packaging` و `/weights` و `/packaging`
- `/admin/collections/pricing` — نمای کلی قیمت

## چک‌لیست دستی
- [ ] **ایجاد محصول**: عنوان فارسی → نامک لاتین خودکار؛ قیمت پایه (تومان) + واحد؛ ذخیره.
- [ ] **ایجاد مدل‌ها (Variant Matrix)**: انتخاب وزن‌ها/بسته‌بندی‌ها → «تولید ماتریس» → ذخیره.
- [ ] **قیمت قفل‌شده**: یک ردیف را «دستی» کن، قیمت تومان وارد کن → بعد از ذخیره دست‌نخورده می‌ماند.
- [ ] **قیمت آزاد**: ردیف «خودکار» → قیمت = محاسبه‌شده (پایه×وزن + بسته‌بندی).
- [ ] **قیمت حراج (offPrice)**: مقدار تومان وارد کن → به ریال ذخیره می‌شود.
- [ ] **افزودن سریع دو محصول**: دو ردیف کامل کن → «ذخیره‌ی همه» → دو محصول + variant ساخته شود؛ لینک «ویرایش کامل».
- [ ] **Paste TSV**: متن tab-separated در ستون عنوان paste کن → ردیف‌ها پر شوند.
- [ ] **کیبورد شیت**: جهت‌ها/Tab/Enter بین سلول‌ها؛ ↓ در آخرین ردیف = ردیف جدید؛ ⌘/Ctrl+S ذخیره؛ ⌘/Ctrl+D تکرار ردیف؛ Esc بستن popover.
- [ ] **فیلتر سازگاری دسته**: در شیت/ماتریس با انتخاب دسته‌ی محصول، فقط وزن/بسته‌بندیِ سازگار (یا سراسری) دیده شود؛ «نمایش همه گزینه‌ها» فیلتر را بردارد.
- [ ] **سازگاری = انتخاب دسته**: در فرم وزن/بسته‌بندی، «سازگار با دسته‌ها» چیپ‌محور از دسته‌های PRODUCT است (نه متن آزاد، نه دسته‌ی POST).
- [ ] **حذف مسدود**: محصولی که خودش یا یکی از مدل‌هایش در سفارش/سبد است → حذف نشود با پیام فارسی.
- [ ] **تکثیر محصول**: کپی **غیرفعال** با SKUهای یکتا و آگاه‌به‌بسته‌بندی.
- [ ] **اعتبارسنجی نامک/SKU**: نامک فارسی دستی رد شود؛ ارقام فارسی نرمال شوند؛ SKU فقط `[A-Z0-9-]`.
- [ ] **ردیف ناموفق در شیت**: خطای ردیفی نشان داده شود، ردیف ویرایش‌پذیر بماند، محصول ناقص ساخته نشود.
- [ ] **build**: `npm run build` سبز.

## ایمنی Seed (مهم)
- `prisma/seed.ts` **فقط برای محیط توسعه** است: همه‌ی داده‌ها را **حذف** و دیتاستِ dev را بازمی‌سازد.
- گارد: اگر `NODE_ENV=production` باشد، seed اجرا نمی‌شود مگر `FORCE_SEED=true` صریحاً تنظیم شود.
- **هرگز** seed را روی دیتابیس production با داده‌ی واقعی اجرا نکنید.
- بعد از تغییر schema / `prisma generate`، اگر سرور dev باز است یک‌بار **ری‌استارت** شود تا کلاینت جدید لود شود.

## ریسک‌های باز / یادداشت‌ها
- صفحه‌ی `/admin/collections/pricing` بدون صفحه‌بندی همه‌ی محصولات + variantها را می‌خواند؛ برای کاتالوگ بزرگ بعداً صفحه‌بندی/تجمیع لازم می‌شود (فعلاً مشکلی نیست).
- `compatibility` به‌صورت `String[]`ِ شناسه‌ی دسته ذخیره می‌شود (نه جدول رابطه‌ای). مهاجرت به رابطه در صورت نیاز در پلن مستند است، اکنون انجام نشده.
- بخش «تصویر/Media»، «سئو» و ویرایشگر «TipTap» در فرم محصول هنوز placeholder هستند (خارج از scope این فاز).
