# QA Report Format

هر checkpoint باید با این فرمت یک QA report بسازد.
نام فایل: `<CHECKPOINT-NAME>-QA.md`

## فرمت

```markdown
# QA Report — <CHECKPOINT-NAME>
**تاریخ:** <ISO date>
**دامنه:** <domain>
**وضعیت کلی:** ✅ / 🟡 / ⚠️

## فایل‌های ساخته/تغییرشده
| فایل | عملیات | توضیح |
|---|---|---|
| `.claude/CLAUDE.md` | ساخته شد | قوانین Claude Code |

## Dependency جدید
- نصب شد: خیر / بله: `<نام>`

## Runtime اپ دشت‌زاد
- دست‌نخورد: ✅ / تغییر کرد: ⚠️ `<توضیح>`

## نتایج تست
| دستور | نتیجه |
|---|---|
| `npx tsc --noEmit` | ✅ pass / ⚠️ N error |
| `npm run claude:map` | ✅ / ⚠️ |
| `npm run claude:pack` | ✅ / ⚠️ |
| `npm run claude:log` | ✅ / ⚠️ |

## خروجی‌های کلیدی
<خلاصهٔ چند خط از خروجی هر دستور>

## محدودیت‌های این CP
- <چه چیزی در scope نبود>

## پیشنهاد CP2
<یک خط، بدون اجرا>
```

## قوانین
- گزارش را کوتاه نگه‌دار (زیر ۸۰ خط).
- نتایج واقعی اجرا را بنویس، نه «احتمالاً کار می‌کند».
- اگر تست اجرا نشد بنویس «اجرا نشد» نه «pass».
