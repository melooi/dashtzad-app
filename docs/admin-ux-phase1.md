# گزارش فاز ۱ — بهبود UI/UX پنل ادمین دشت‌زاد

تاریخ: ۱۴۰۵/۰۴/۰۵ (۲۰۲۶-۰۶-۲۶)
محدوده: فقط `/admin`، `src/components/admin`، `src/lib/admin`. بدون تغییر Prisma، بدون Payload، بدون فرانت عمومی/دیزاین‌سیستم.

## چه چیزی اصلاح شد

### ۱) Empty Stateهای دو‌حالته
کامپوننت مشترک `AdminListEmptyState` ساخته شد با دو حالت:
- **بدون داده** → دلیل خالی‌بودن + توضیح کاربرد + CTA افزودن.
- **نتیجهٔ فیلتر خالی** → دکمهٔ «پاک کردن فیلترها» + اقدام اصلی.

اعمال‌شده روی: کوپن‌ها، محصولات، مقاله‌ها، سفارش‌ها، پیام‌های تماس.
- سفارش‌ها و پیام‌های تماس بدون CTA (این‌ها از سمت کاربر ساخته می‌شوند، نه ادمین).
- **رسانه**: از قبل Empty State دو‌حالته‌ی اختصاصی (`MediaEmptyState`) داشت؛ دست‌نخورده ماند (الگوی مرجع).

### ۲) استانداردسازی Save Bar
`AdminSubmitBar` به سه حالت ارتقا یافت:
- ذخیره‌شده (کم‌رنگ) / تغییرات ذخیره‌نشده / **خطا** (تعداد خطا + دکمهٔ «رفتن به اولین خطا»).
- گزینهٔ انصراف انعطاف‌پذیر شد: یا لینک (`cancelHref`) یا دکمهٔ «بازگردانی» (`onCancel`) برای صفحات تک‌صفحه‌ای مثل گلوبال‌ها.
- نوار درون‌خطیِ موازی در `GlobalForm` حذف و با همین کامپوننت مشترک جایگزین شد.

### ۳) Form Navigator برای فرم‌های طولانی
کامپوننت مشترک `AdminFormNavigator` ساخته شد (ناوبری چسبان با IntersectionObserver، فقط در `lg+`).
`AdminFormSection` یک `id` اختیاری گرفت تا anchorها کار کنند.
اعمال‌شده روی: **ویرایش مقاله** (`ArticleForm`)، **پروندهٔ محتوایی** (`CaseFileForm`)، **همهٔ گلوبال‌ها** (`GlobalForm` — شامل هدر، فوتر، تماس، سئو…).

### ۴) Danger Zone کنترل‌شده
`AdminDangerZone` پیش‌فرض **جمع‌شده** شد و با کلیک باز می‌شود (دیگر باکس قرمز غالب نیست).
تأیید حذف به «حذف دائمی» تغییر کرد و propهای اختیاری `isPublished` + `onUnpublish` اضافه شد تا «اول خارج از انتشار، بعد حذف» پیشنهاد شود.

### ۵) Preview / View On Site یکدست
`AdminViewOnSiteButton` سه حالت گرفت: **مشاهده در سایت** (منتشرشده) / **پیش‌نمایش** (پیش‌نویس) / **غیرفعال با توضیح** (بدون نشانی عمومی).
اعمال‌شده روی صفحات ویرایش مقاله و پرونده: پیش‌نویس‌ها حالا دکمهٔ غیرفعالِ گویا دارند به‌جای نبودِ دکمه.

### ۶) پولیش سایدبار (کم‌ریسک، بدون تغییر معماری)
- همهٔ برچسب‌ها `truncate` + tooltip (رفع «سوالات …» و «متن صف…»).
- گروهِ شاملِ مسیرِ فعال، تیتر پررنگ‌تر دارد (بهتر دیده می‌شود).

## چه چیزی عمداً باقی ماند (بدهی UX برای فازهای بعد)

- **Empty State**: هنوز روی دیگر لیست‌ها اعمال نشده (دسته‌بندی‌ها، بنرها، منوها، ریدایرکت‌ها، دیدگاه‌ها، نقدها، دستور پخت‌ها، پیشنهادها). الگو آماده است؛ فقط باید مثل کوپن‌ها سیم‌کشی شود.
- **Form Navigator**: روی `ProductForm`، `CouponForm` و فرم‌های وزن/بسته‌بندی اعمال نشده. الگو آماده است.
- **unpublish-first در Danger Zone**: propها اضافه شد ولی هنوز هیچ فرمی `onUnpublish` پاس نمی‌دهد (نیازمند یک server action سبک برای تغییر وضعیت به پیش‌نویس — بدون تغییر اسکیما).
- **سایدبار**: تعداد آیتم‌ها زیاد است؛ برای فاز بعد پیشنهاد می‌شود گروه‌بندی فشرده‌تر یا قابلیت «سنجاق‌کردن میان‌برهای پرکاربرد» بررسی شود (نیازمند تصمیم طراحی، نه این فاز).
- **AdminViewOnSiteButton**: برای هدر/فوتر/تماس/قوانین/FAQ یک «پیش‌نمایش صفحهٔ عمومی مرتبط» می‌تواند اضافه شود (نیازمند نگاشت گلوبال→URL عمومی).

## پیشنهاد فاز ۲ (فقط پیشنهاد، اجرا نشده)
1. تکمیل Empty State روی همهٔ لیست‌های باقی‌مانده.
2. server actionهای سبک «تغییر وضعیت انتشار» برای فعال‌کردن unpublish-first.
3. اعمال Form Navigator روی فرم محصول و کوپن.
4. بازنگری معماری سایدبار (سنجاق میان‌بر / جستجوی ناوبری) به‌صورت یک طرح جدا.
